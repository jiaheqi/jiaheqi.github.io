背景

为了实现自己的 github pages 博客页在在谷歌搜索中可见。

实现

站点提交到 Google

登录到谷歌控制台 谷歌控制台提交地址


使用推荐的 html 方式，下载 html 文件后上传的github pages 工程目录下
然后点击验证

但是这种方式我一直验证失败，所以更换了网域的形式。

后续找到了链接验证无法成功原因，是因为我把下载下来的 html 文件放在了 source 目录下，然后 hexo 编译后在 public 目录下生成的文件，会被 hexo 渲染，原本的内容只有google-site-verification: googlef1ecefe81f454c0d.html,被 hexo 渲染后会多很多标签和内容，所以要保证 html 不被hexo 渲染。
如何实现某些文件不被 hexo 渲染呢？参考：
github pages 实现某些文件不被 hexo 渲染

然后改用了新的网域方式，在阿里云的 dns 解析中添加 txt 类型记录


验证成功之后需要一天的时间才会生效
